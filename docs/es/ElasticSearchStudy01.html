    基本数据类型

    wangruiyu 2020/12/4 ElasticSearch

    # 数据类型

    官网地址 (opens new window)

    # 核心数据类型 （Core data types）

    官网地址 (opens new window)

    • string 串 (opens new window)

      text (opens new window) 和 keyword (opens new window)

    • Number 数字 (opens new window)

      long，integer，short，byte，double，float，half_float，scaled_float

    • Date 日期 (opens new window)

      date

    • Date nanoseconds 日期纳秒 (opens new window)

      date_nanos

    • Boolean 布尔型 (opens new window)

      boolean

    • Binary 二进制 (opens new window)

      binary

    • Range 范围 (opens new window)

      integer_range，float_range，long_range，double_range，date_range，ip_range

    # 复杂数据类型（Complex data types）

    官网地址 (opens new window)

    • Object 对象 (opens new window)

      object 用于单个JSON对象

    • Nested 嵌套对象 (opens new window)

      nested 用于JSON对象数组

    # 地理数据类型 （Geo data types）

    官网地址 (opens new window)

    • Geo-point 地理位置 (opens new window)

      geo_point 纬度/经度积分

    • Geo-shape 地理形状 (opens new window)

      geo_shape 用于多边形等复杂形状

    # 专用数据类型 （Specialised data types）

    官网地址 (opens new window)

    • IP (opens new window)

      ip 用于IPv4和IPv6地址

    • Completion data type 完成数据类型 (opens new window)

      completion 提供自动完成建议

    • Token count (opens new window)

      token_count 计算字符串中令牌的数量

    • mapper-murmur3 (opens new window)

      murmur3 在索引时计算值的哈希并将其存储在索引中

    • mapper-annotated-text (opens new window)

      annotated-text 索引包含特殊标记的文本（通常用于标识命名实体）

    • Percolator (opens new window)

      接受来自query-dsl的查询

    • Join (opens new window)

      为同一索引内的文档定义父/子关系

    • Rank feature 排名功能 (opens new window)

      记录数字功能以提高查询时的点击率。

    • Rank features 等级特征 (opens new window)

      记录数字功能以提高查询时的点击率。

    • Dense vector密集矢量 (opens new window)

      记录浮点值的密集向量。

    • Sparse vector 稀疏矢量 (opens new window)

      记录浮点值的稀疏向量。

    • Search -as-you-type 按类型搜索 (opens new window)

      针对查询进行优化的类文本字段，以实现按需输入完成

    • Alias 别名 (opens new window)

      为现有字段定义别名。

    • Flattened 展平 (opens new window)

      允许将整个JSON对象索引为单个字段。

    • Shape 形状 (opens new window)

      shape 对于任意笛卡尔几何。

    • Histogram 直方图 (opens new window)

      histogram 用于百分位数聚合的预聚合数值。

    • constant keyword 常数关键字 (opens new window)

      keyword当所有文档具有相同值时的情况的 专业化。

    # 数组 （Arrays）

    官网地址 (opens new window)

    在Elasticsearch中，数组不需要专用的字段数据类型。默认情况下，任何字段都可以包含零个或多个值，但是，数组中的所有值都必须具有相同的数据类型。请参阅数组 (opens new window)。

    # 多领域 (Multi-fieds)

    官网地址 (opens new window)

    为不同的目的以不同的方式对同一字段建立索引通常很有用。例如，一个string字段可以映射为text用于全文搜索的字段，也可以映射为keyword用于排序或聚合的字段。或者，您可以使用standard分析仪 (opens new window)， english (opens new window)分析仪和 french分析仪 (opens new window)索引文本字段。

    这是多领域的目的。大多数数据类型通过fields (opens new window)参数支持多字段。

    # Rest风格说明

    一种软件架构风格,而不是标准,只是提供了一组设计原则和约束条件。它主要用于客户端和服务器交互类的软件。基于这个风格设计的软件可以更简洁,更有层次,更易于实现缓存等机制。

    method URL 描述
    PUT localhost:9200/索引名称/类型名称/文档Id 创建/修改文档（指定文档Id）
    POST localhost:9200/索引名称/类型名称 创建文档（随机文档Id）
    POST localhost:9200/索引名称/类型名称/文档Id/_update 修改文档
    DELETE localhost:9200/索引名称/类型名称/文档Id 删除文档
    GET localhost:9200/索引名称/类型名称/文档Id 查询文档 通过文档Id
    POST localhost:9200/索引名称/类型名称/_search 查询所有文档
